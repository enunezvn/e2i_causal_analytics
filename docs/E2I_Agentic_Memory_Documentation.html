Memory Type Specifications
As defined in Deep Agentic Memory architecture
Memory Type Function Technology
Short-Term (Working) Holds current context, scratchpad, and immediate message history Redis via LangGraph MemorySaver checkpointer + In-Context Prompting
Episodic (Long-Term) Stores experiences: "What did I do?" "What happened yesterday?" Supabase (Postgres + pgvector). User/AI interaction traces.
Semantic (Long-Term) Stores facts: "What is the relationship between Project X and User Y?" FalkorDB + Graphity. Extractor node updates this graph.
Procedural (Long-Term) Stores skills: "How did I solve this error last time?" Supabase. Vector store of successful tool call sequences.

Detailed breakdown of each memory type, its purpose, and implementation.

‚ö°
Working Memory
Short-Term / Active Context
Redis LangGraph MemorySaver

Holds the active reasoning context with automatic state persistence across workflow steps.

  • Session state (user context, filters)
  • Message history (last N for in-context prompting)
  • Evidence board (accumulated during investigation)
  • Scratchpad (intermediate computations)
  • 24-hour TTL with automatic cleanup
üìñ
Episodic Memory
Long-Term / Experiences
Supabase pgvector

Stores User/AI interaction traces. Answers: "What did I do?" "What happened yesterday?"

  • User queries and responses
  • Agent actions and outcomes
  • System events and errors
  • Feedback and corrections
  • Vector similarity search (1536 dims)
üï∏Ô∏è
Semantic Memory
Long-Term / Facts & Relationships
FalkorDB Graphity

Stores facts as triplets. Answers: "What is the relationship between X and Y?"

  • Entity nodes (Patient, HCP, Brand, Region)
  • Relationship edges (TREATED_BY, CAUSES)
  • Causal chain traversal
  • Auto-extraction via Graphity LLM
  • Cache layer in Supabase
üîß
Procedural Memory
Long-Term / Skills & Patterns
Supabase pgvector

Stores successful tool sequences. Answers: "How did I solve this error last time?"

  • Tool call sequences (ordered steps)
  • Query patterns that worked
  • Error recovery procedures
  • Few-shot examples for prompting
  • Success rate tracking

üöÄ Setup Guide

Get the E2I Agentic Memory System running on your local machine.

Install Redis (Working Memory)

Redis serves as the short-term working memory with LangGraph checkpointing.

# macOS
brew install redis
brew services start redis

# Ubuntu/Debian
sudo apt-get install redis-server
sudo systemctl start redis

# Docker
docker run -p 6379:6379 -d redis
Install FalkorDB (Semantic Memory)

FalkorDB provides graph-based semantic memory with Graphity auto-extraction.

# Docker (recommended) - use different port to avoid conflict
docker run -p 6380:6379 -it --rm falkordb/falkordb:latest

# Verify connection
redis-cli -p 6380 PING
Set Environment Variables

Configure connections to all memory backends.

# Redis (Working Memory)
export REDIS_URL="redis://localhost:6379"

# FalkorDB (Semantic Memory)
export FALKORDB_HOST="localhost"
export FALKORDB_PORT="6380"

# Supabase (Episodic + Procedural Memory)
export SUPABASE_URL="https://your-project.supabase.co"
export SUPABASE_ANON_KEY="your-anon-key"

# LLM Services
export ANTHROPIC_API_KEY="your-anthropic-key"
export OPENAI_API_KEY="your-openai-key"
Install Python Dependencies

Install required packages for all memory backends.

pip install redis falkordb supabase \
            langgraph langgraph-checkpoint-redis \
            anthropic openai pydantic pyyaml
Apply Database Schema

Run the SQL schema in Supabase to create episodic and procedural memory tables.

# In Supabase SQL Editor, run:
# 001_agentic_memory_schema.sql

# This creates:
# - episodic_memories (with pgvector)
# - procedural_memories (with pgvector)
# - semantic_memory_cache
# - cognitive_cycles
# - learning_signals
Run Cognitive Cycle

Test the system with a sample query.

from e2i_agentic_memory.cognitive_workflow import run_cognitive_cycle
from e2i_agentic_memory.memory_backends import get_langgraph_checkpointer

# Get Redis-backed checkpointer
checkpointer = get_langgraph_checkpointer()

# Run cognitive cycle
result = await run_cognitive_cycle(
    user_query="Why did Kisqali adoption increase in the Northeast?",
    user_id="analyst_001",
    checkpointer=checkpointer
)

print(f"Response: {result['synthesized_response']}")
print(f"Confidence: {result['confidence_score']:.0%}")
print(f"Facts Learned: {len(result['new_facts'])}")
