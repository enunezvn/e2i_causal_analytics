üõ°Ô∏è Data Leakage Prevention Measures
‚è∞ Temporal Isolation
7-day gaps between splits prevent information bleeding across time boundaries. Future data cannot influence past predictions.
üë§ Patient-Level Isolation
Each patient exists in exactly ONE split. All their events stay together. Tracked in ml_patient_split_assignments table.
üìê Preprocessing Isolation
Scaling statistics computed ONLY on training data, stored in ml_preprocessing_metadata with validation constraint.
üéØ Outcome Leakage Prevention
features_available_at_prediction tracks what data was known at prediction time. Outcomes never used as features.

Python - Load Training Data Only

# Using Supabase client from supabase import create_client client = create_client(url, key) # Get training data only - no leakage! train_journeys = client.table('patient_journeys')\ .select('*')\ .eq('data_split', 'train')\ .execute() # Or use the view train_data = client.table('v_train_patient_journeys')\ .select('*')\ .execute()

SQL - Get Preprocessing Stats

-- Get preprocessing metadata (computed on train only) SELECT * FROM get_preprocessing_stats( 'your-split-config-uuid' ); -- Run leakage audit SELECT * FROM run_leakage_audit( 'your-split-config-uuid' );

SQL - Check Split Statistics

-- View split statistics summary SELECT * FROM v_split_statistics; -- Verify no patients in multiple splits (should be 0) SELECT patient_id, COUNT(DISTINCT data_split) as split_count FROM patient_journeys GROUP BY patient_id HAVING COUNT(DISTINCT data_split) > 1;
